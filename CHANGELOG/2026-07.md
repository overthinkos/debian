# Changelog — 2026-07

### 2026-07-01 — test: assert welded commands resolve project-less on the .deb install (check-debian-debootstrap-vm)

Added a `welded-command-project-less` check step to the `check-debian-debootstrap-vm`
bed. The existing `localpkg-deb-autoresolve` step asserted only `charly version` (a
core command that always resolves), so it could not catch the packaging gap where the
`.deb` shipped only the binary without the welded `command:<word>` plugins. The new
step asserts the baked plugins are present at `/usr/lib/charly/plugins`
(`plugin-doctor`/`plugin-vm`/`plugin-clean` + `plugin-doctor.providers`) AND that a
project-less `charly doctor` actually dispatches through the baked `command:doctor`
plugin (`WELDED-CMD-PROJECT-LESS-OK`) — it FAILS on a package that shipped only the
binary. Pairs with the `pkg/debian` `.deb` plugin-baking fix.
